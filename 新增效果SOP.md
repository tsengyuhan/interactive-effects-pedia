# 新增效果 SOP（給 AI 的標準作業流程）

收到「讀 templates/某效果.md，照新增效果SOP.md 加上這個效果」時，依序執行：

## 流程

1. **讀需求單**，不清楚處先向使用者確認再動工。
2. **建資料夾** `effects/<slug>/`（slug：小寫英文連字號）：
   - `index.html`——統一骨架（抄任一既有效果，改 title）
   - `effect.js`——效果本體，必須使用 `Shell.init({id})` 外殼 API
     （參考 `assets/shell.js` 開頭的 API 註解與既有效果的用法）
3. **函式庫規則**：新函式庫一律下載進 `libs/<名稱>/` 並建 `VERSION.md`
   記錄版本與來源網址。**禁用 CDN**。需要 ESM 的效果用
   `<script type="module">`（此類效果無法以 file:// 開啟，屬正常）。
4. **登錄** `effects/effects.js` 加一筆，欄位契約見該檔註解。
   principle 條列總計 200 字內；requirements 如實寫（含攝影機/麥克風、
   建議瀏覽器、效能備註）；offline 如實標註。
5. **縮圖**：擷取效果代表畫面存為 `effects/<slug>/thumb.png`（建議 800×500）。
   無法截圖時先略過（首頁有佔位樣式），並在回報中註明。
6. **驗證後才能回報完成**：
   - `start.bat` 開啟，首頁卡片與篩選正常、效果行為符合需求單
   - 中斷網路重新整理，效果仍可用（除非需求單明示需連網）
   - 攝影機/麥克風類：封鎖權限後重新整理，出現繁中錯誤提示、無白屏
7. **鐵律：不得修改其他效果資料夾內的任何檔案。** 共用檔（shell、首頁）
   原則上不動；非改不可時，必須先說明理由取得同意。
8. **Commit**：`feat: <效果名稱>效果`。
