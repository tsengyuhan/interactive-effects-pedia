# 互動效果百科 — 設計規格

日期：2026-06-13
狀態：已與使用者逐項確認

## 1. 目的

集結各種互動網頁效果的展示網站，用途：

1. 思考互動展項、與同事朋友討論時，有可直接操作的範例佐證技術可行性
2. 個人靈感資料庫
3. 透過固定的需求模板，讓 AI 能快速理解並新增效果

核心精神：每個項目是「單一效果」的呈現，不是完整互動作品。

## 2. 技術方案

**純靜態網站、零建置工具**（已與方案 B「Vite 建置」、方案 C「框架 SPA」比較後選定）：

- 原生 HTML / CSS / JS，沒有 npm、沒有 build step
- 第三方函式庫（MediaPipe 等）**一律下載進專案 `libs/` 資料夾**，不用 CDN——同時達成離線執行與版本鎖定
- push 到 GitHub 即可由 GitHub Pages 直接上線

選定理由：

- AI 新增效果的路徑最短（建資料夾＋登錄一筆），不碰工具鏈
- 效果之間完全隔離，一個壞了不影響其他
- 每個效果是自足的原生 JS，之後搬進正式展覽專案直接複製即可
- 整包資料夾複製到展場電腦就能離線執行，無需建置產物

## 3. 目錄結構

```
互動效果百科/
├── index.html              ← 首頁（卡片牆）
├── assets/                 ← 共用樣式、首頁 JS、效果頁外殼（shell）
├── libs/                   ← 下載進專案的第三方函式庫與模型檔
├── effects/
│   ├── effects.js          ← 中央登錄檔（唯一資料來源）
│   ├── <效果slug>/          ← 每個效果一個資料夾，完全自足
│   │   ├── index.html
│   │   ├── effect.js
│   │   └── thumb.png       ← 卡片縮圖（靜態截圖）
│   └── ...
├── templates/
│   └── 新增效果需求.md      ← 使用者填寫的需求模板
├── 新增效果SOP.md           ← 給 AI 的標準作業流程
├── start.bat               ← 雙擊啟動本機伺服器（展場／攝影機麥克風權限用）
└── docs/superpowers/specs/ ← 設計文件
```

關鍵決定：

- **登錄檔用 `.js`（`window.EFFECTS = [...]`）而非 `.json`**：雙擊直接開 HTML（`file://`）時瀏覽器禁止 fetch JSON，JS 檔不受限，保住「複製整包、雙擊就能跑」。
- **附 `start.bat`**：攝影機／麥克風效果在部分瀏覽器設定下 `file://` 拿不到裝置權限，雙擊腳本啟動本機伺服器即可解決。純滑鼠效果雙擊 HTML 即可。

## 4. 中央登錄檔欄位（每個效果一筆）

| 欄位 | 說明 |
|---|---|
| id | 資料夾名稱（slug） |
| title | 效果名稱（繁中） |
| category | 分類：`網頁互動`／`身體動作`／`聲音互動`（可擴充） |
| description | 一句話描述 |
| instructions | 操作說明（一兩句） |
| tech | 主要使用技術標籤陣列（如 `Canvas 2D`、`MediaPipe Hands`） |
| principle | 白話原理，條列式陣列，總計 200 字內 |
| requirements | 軟硬體規格（攝影機／麥克風需求、建議瀏覽器、效能備註） |
| offline | 連線需求：`true`＝完全離線可用；`false` 時附 `offlineNote` 說明需網路的原因 |
| hasParams | 是否有可調參數面板 |

## 5. 首頁

視覺風格走編輯／刊物感（2026-06-14 改版）：

- 配色：暖米白紙感底色（`--bg #f4f2ea`）＋近黑墨色文字，髮絲線用暖灰；淺色主題。
- 字體：標題用襯線（Playfair Display ＋ Noto Serif TC，離線退回 Georgia／系統襯線）；內文與標籤用無襯線。Google Fonts 線上載入、離線優雅降級。
- Hero：斜體英文小標（kicker）＋襯線大標＋一段內文。文字內容在 `index.html`，字級在 `assets/home.css` 的 `.hero-*`。
- 卡片式網格：縮圖、效果名稱、meta 列（分類／離線狀態，小型大寫＋字距，不用 emoji 徽章）。一進首頁即顯示全部卡片，不分區、無分類標題。
- 頂部分類籤列（chip：全部／網頁互動／身體動作／聲音互動），點擊即時篩選；篩選用 FLIP 動畫：不符合者淡出移出佈局，符合者平滑往前補位，再次符合則淡入（Web Animations API）。
- 無頂部選單列（logo／站名導覽列已移除）。
- 介面繁體中文；響應式，手機可順暢瀏覽。
- 點卡片進入該效果頁。

## 6. 效果頁共用外殼（shell）

每個效果頁載入共用的 `assets/` 外殼程式，效果本身只負責畫自己的 canvas：

- 畫面主體：效果全螢幕呈現
- 左上角：返回首頁鍵
- 右側可收合資訊面板，內容全部讀自中央登錄檔：
  - 操作說明（預設展開，數秒後自動收起）
  - 主要使用技術（標籤列）
  - 白話原理（條列）
  - 軟硬體規格
  - 連線需求
  - 可調參數面板（選配，`hasParams` 為真才顯示）
- 錯誤處理：攝影機／麥克風權限被拒或裝置不存在時，顯示繁中友善提示與重試指引，不得白屏

## 7. 初始三個效果

三個效果各驗證一條技術路徑，全部完全離線可跑。

### 7.1 水墨筆觸（網頁互動／Canvas 2D）

- 畫布底為程式產生的宣紙紋路（不用外部圖片）
- 長按／拖曳出墨跡，具濃淡暈染動畫：落筆濃、邊緣漸滲；拖快出飛白、拖慢墨積暈染
- 參數面板：墨色濃度、暈染擴散程度、筆刷大小、筆墨顏色
- 右上角清空畫布按鈕
- 重點：水墨質感與暈染動畫

### 7.2 手指取景框（身體動作／MediaPipe Hands）

- 同時偵測兩隻手；每隻手拇指與食指張開呈約 90 度（L 字手勢，含角度容差）時視為有效角點，角點取虎口位置
- 兩手角點作為矩形對角，撐出「取景框」；雙手移動時框即時縮放平移
- 框內的即時攝影機畫面套用特效；參數面板可即時切換：負片／馬賽克（兩種都實作）
- 畫面繪出手部偵測節點與框線，讓觀者理解追蹤狀態
- 偵測不到兩個有效 L 手勢時框淡出並顯示提示
- MediaPipe 函式庫與模型檔放入 `libs/`，完全離線
- 效能備註：舊機器約 15–25 fps，如實標註於軟硬體規格
- 重點：畫面即時改變、手勢偵測

### 7.3 聲音漣漪（聲音互動／Web Audio API）

- 畫面為俯視池塘水面，採高度場水波模擬（water ripple 演算法）：漣漪互相干涉、反彈、自然衰減
- 音量 → 漣漪大小（投入能量）；音高 → 漣漪持續時間（映射方向實作時可對調，取直覺者）
- 畫面角落固定顯示即時音量（dB）與音高（Hz）數值，附即時量條
- 音高偵測用自相關演算法，Web Audio 原生實作，離線可用
- 重點：聲音偵測數據可視、寫實物理漣漪

## 8. 需求模板與 AI 新增流程

### 8.1 `templates/新增效果需求.md` 欄位

- 效果名稱
- 分類（既有三類或新分類）
- 一句話描述
- 互動方式（使用者做什麼）
- 視覺呈現描述
- 參考連結或影片（選填）
- 需要可調參數嗎（要的話列出哪些）
- 連線需求預期（預設離線）
- 備註

### 8.2 `新增效果SOP.md`（給 AI 的固定流程）

1. 讀使用者填好的需求檔
2. 在 `effects/` 建立新資料夾並實作（自足、原生 JS、共用外殼）
3. 新函式庫一律下載進 `libs/`，禁用 CDN
4. 在 `effects.js` 登錄一筆完整 metadata
5. 產生縮圖 `thumb.png`（截圖）
6. 驗證：本機伺服器開啟正常、離線（中斷網路）可跑、權限拒絕時提示正常
7. 禁止修改其他效果的程式碼
8. commit

使用者固定句式：「讀 templates/某某效果.md，照新增效果SOP.md 加上這個效果」。

## 9. 部署

- git repo，push 到 GitHub，GitHub Pages 從 main 分支根目錄發布
- 之後每次新增效果 push 即自動更新線上版

## 10. 驗收條件

1. 首頁顯示三張效果卡片，分類籤篩選正確，每卡有縮圖、分類標籤、離線徽章
2. 三個效果依第 7 節行為正常運作；參數面板即時生效；水墨清空鍵有效
3. 中斷網路後，整站（含 MediaPipe 手部效果）仍可完整執行
4. 拒絕攝影機／麥克風權限時顯示繁中提示，無白屏
5. 雙擊 `index.html`（`file://`）可瀏覽首頁與水墨效果；`start.bat` 啟動後三個效果皆可用
6. `templates/新增效果需求.md` 與 `新增效果SOP.md` 存在且內容完整，照 SOP 可在不動其他效果的前提下新增效果
7. GitHub Pages 線上網址可開啟並正常操作（攝影機／麥克風在 HTTPS 下正常請求權限）

## 11. 不做的事（YAGNI）

- 不做網站內的需求填寫表單（模板用 Markdown 檔）
- 不做懸停影片預覽或卡片內即時預覽（縮圖為靜態截圖）
- 不做後端、資料庫、使用者系統
- 不做建置工具鏈
